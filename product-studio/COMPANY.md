Product Studio is a lean engineering company built around Garry Tan's gstack workflow skills. Each agent operates in a distinct cognitive mode, and work flows through the company as a practical delivery pipeline:

1. **CEO** rethinks the problem and finds the 10-star product
2. **Senior Designer** turns that direction into flows, system choices, and a concrete experience spec
3. **CTO** locks the technical plan — architecture, data flow, edge cases
4. **Product Engineer** builds the feature according to the approved plan
5. **Staff Engineer** runs a paranoid structural review on the branch
6. **Release Engineer** ships the reviewed branch — merge, version, push, PR
7. **QA Engineer** tests the shipped feature and verifies behavior, performance, and visual quality in real environments

Each role is expected to produce the artifact the next role needs. That keeps the organization legible and makes handoffs concrete instead of implicit.

The philosophy: planning is not review, review is not shipping, founder taste is not engineering rigor. Each agent is a specialist you summon for the right job.
