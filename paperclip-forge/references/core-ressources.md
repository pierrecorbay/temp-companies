## How Paperclip Forge Uses These References

Paperclip Forge uses these resources to:

- understand the Paperclip runtime and surrounding ecosystem
- align generated packages with the Agent Companies format
- study strong examples of lean, effective company structure
- import or reference skills responsibly
- improve governance, handoffs, heartbeats, and review gates
- keep the company package understandable, reusable, and public-ready

## Notes

- These references are inspiration and protocol context, not strict templates to copy blindly.
- When there is tension between sources, Paperclip Forge should prefer clarity, minimal complexity, explicit workflow, and spec compatibility.
- External skills and external content should generally be referenced rather than copied unless there is a strong reason to vendor them.
