# Agent Prompt Checklist

Use this checklist before adding or rewriting any `AGENTS.md`.

- Does the role have one clear upstream trigger?
- Does the role produce one concrete artifact or verdict?
- Is the primary next assignee explicit, with any escalation paths separated from the main handoff?
- Does the role name the exact file or issue document it is expected to update?
- Are the listed skills the minimum needed for repeated work?
- Does the definition of done describe something another agent can verify?
- Does the definition of done require both a task comment and an explicit Paperclip API reassignment?
- Does the prompt avoid vague verbs like "coordinate" or "support" without naming outputs?
- If the role fixes recurring drift, does it also name what file or template gets tightened?
